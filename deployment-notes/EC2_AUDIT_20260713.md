## TLS and Domain Findings

Certificate served on `443`:

- Subject: `CN=21-holdem.com`
- Issuer: Let's Encrypt `YR1`
- Valid from: `2026-05-31 18:00:39`
- Expires: `2026-08-29 18:00:38`
- SANs:
  - `21-holdem.com`
  - `www.21-holdem.com`

Findings:

- Critical: `https://api.21-holdem.com` fails trust validation because the certificate does not include `api.21-holdem.com`.
- Critical: `http://api.21-holdem.com` responds over plaintext HTTP instead of redirecting to HTTPS.
- Medium: `http://21-holdem.com` and `http://www.21-holdem.com` correctly redirect to HTTPS.
- Medium: HTTPS responses do not include common hardening headers:
  - `Strict-Transport-Security`
  - `Content-Security-Policy`
  - `X-Frame-Options`
  - `X-Content-Type-Options`
  - `Referrer-Policy`
- Informational: Apache exposes `Server: Apache/2.4.52 (Ubuntu)`.

## Application Reachability

Observed:

- `https://21-holdem.com` returns `200`.
- `https://www.21-holdem.com` returns `200`.
- `https://21-holdem.com/socket.io/?EIO=4&transport=polling` returns a Socket.IO handshake.
- `https://21-holdem.com/api/v1` returns `404`, which is acceptable if the API has no root route.
- `https://api.21-holdem.com/api/v1` fails due to the certificate mismatch.

Risk:

- If any frontend, mobile, admin, webhook, or third-party callback uses `https://api.21-holdem.com`, it will fail TLS validation.
- If clients fall back to `http://api.21-holdem.com`, credentials/session data could traverse plaintext.

## Priority Fixes

Repo remediation prepared:

- `scripts/ec2-remediate-21holdem-apache.sh`
- Must be run on the EC2 instance as root.
- Backs up Apache config before changes.
- Renews the Let's Encrypt cert with `api.21-holdem.com`.
- Forces HTTP to HTTPS.
- Adds baseline hardening headers.
- Reloads Apache only after `apache2ctl configtest` passes.

1. Fix `api.21-holdem.com` TLS.
   - Add `api.21-holdem.com` to the Let's Encrypt certificate or remove the hostname if unused.
   - Suggested Certbot shape on EC2:
     - `sudo certbot --apache -d 21-holdem.com -d www.21-holdem.com -d api.21-holdem.com`
   - Verify:
     - `curl -I https://api.21-holdem.com`

2. Force HTTPS for `api.21-holdem.com`.
   - Add Apache rewrite/redirect for API vhost on port `80`.
   - Plain HTTP should return `301` to `https://api.21-holdem.com/...`.

3. Restrict SSH.
   - Security group should allow port `22` only from trusted admin IPs, or replace direct SSH with AWS Systems Manager Session Manager.
   - Confirm no `0.0.0.0/0` or `::/0` SSH rule exists.

4. Add security headers in Apache.
   - Minimum recommended headers:
     - `Strict-Transport-Security: max-age=31536000; includeSubDomains`
     - `X-Content-Type-Options: nosniff`
     - `X-Frame-Options: DENY`
     - `Referrer-Policy: strict-origin-when-cross-origin`
   - Add CSP after checking current scripts/assets to avoid breaking the app.

5. Confirm backups.
